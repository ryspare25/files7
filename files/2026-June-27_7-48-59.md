Common reverse proxy features
Load balancing – distribute requests across servers.
TLS/SSL termination – handle HTTPS encryption so backend servers don't each need to perform it.
Caching – serve frequently requested content directly.
Compression – reduce response size.
Rate limiting – slow abusive clients.
Web Application Firewall (WAF) – block malicious requests.
Authentication – require login before forwarding traffic.
Health checks – stop sending requests to unhealthy servers.
