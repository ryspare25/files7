The slide summarizes the key differences between a forward proxy and a reverse proxy: Aspect Forward Proxy Reverse Proxy Protects / represents The client The server Position Between the client and the internet Between users and backend servers Who knows it exists? Clients usually configure or knowingly use it Users often do not realize it is there Typical purpose Anonymity, filtering, access control Load balancing, caching, security, TLS termination Example VPN / web proxy Nginx, Cloudflare, AWS ALB The simplest distinction A forward proxy works for users. Your computer sends its request to the proxy first; the proxy then visits the website on your behalf. Your browser → Forward Proxy → Internet / Website To the website, the request appears to come from the proxy rather than directly from you. A reverse proxy works for servers. A visitor sends a request to a public domain, but the reverse proxy receives it first and decides which internal server should handle it. User → Reverse Proxy → App Server 1 / App Server 2 / App Server 3 The user may think they are talking directly to one website, while the reverse proxy is quietly distributing traffic, caching content, blocking attacks, and hiding the backend infrastructure. Example Suppose a company has three application servers: app-server-1 app-server-2 app-server-3 Instead of exposing all three directly to the public internet, it exposes only: www.company.com A reverse proxy such as Nginx receives every request to www.company.com, then forwards each request internally to one of those servers. That gives the company: one public entry point; load balancing; backend servers hidden from direct public access; centralized HTTPS certificates; caching; rate limiting and web-application-firewall rules; easier maintenance and scaling. By contrast, if an employee uses a forward proxy: Employee laptop → Company forward proxy → YouTube / GitHub / other websites the company can control outbound browsing, block prohibited sites, log traffic, or make external websites see the company proxy’s IP address rather than the employee’s device IP. Memory shortcut Forward proxy: “I hide the client.” Reverse proxy: “I hide the server.”
