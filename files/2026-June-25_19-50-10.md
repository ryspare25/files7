Direct Answer
A recursive resolver (or DNS recursor) is a server that acts as an intermediary between your device and the rest of the Domain Name System (DNS). When you type a URL into your browser, it is the recursive resolver's job to hunt down the correct IP address by tracking it down across other DNS servers.
How It Works
Receives Request: Your device asks the recursive resolver for a website's IP address.
Searches Cache: It checks its own memory first to see if it visited that site recently.
Queries Others: If not cached, it asks the Root, TLD, and Authoritative servers in sequence.
Delivers Answer: It returns the final IP address back to your browser so the website can load.
Key Analogy
Think of a recursive resolver like a librarian. Instead of making you search through multiple floors and catalog systems yourself, you ask the librarian for a book. The librarian goes into the stacks, finds exactly what you need, and brings it back to you.
