A proxy is a server that sits between two parties and passes requests and responses between them.
Instead of your device communicating directly with a website/server, the flow becomes:
You → Proxy → Website/Server
You ← Proxy ← Website/Server
The proxy can inspect, filter, cache, block, modify, or route the traffic.
Why use a proxy?
A proxy can provide:
Security — blocks dangerous sites, malware, or unauthorized requests.
Access control — limits which websites or services users can reach.
Caching — stores frequently requested content so later requests can be served faster.
Logging and monitoring — records traffic for auditing, troubleshooting, or policy enforcement.
Anonymity/privacy — hides the client’s IP address from the destination server, depending on configuration.
Traffic management — controls bandwidth, balances requests, or applies rules to network traffic.
The two main types
1. Forward proxy
A forward proxy represents the client.
Employee computer → Forward Proxy → Internet website
The destination website sees the proxy’s address rather than the employee’s device directly.
Example: A company requires employees to access the internet through a corporate proxy. The proxy can block social media, scan downloads, cache common websites, and log browsing activity.
Think: “I am going outward to the internet, and the proxy goes on my behalf.”
2. Reverse proxy
A reverse proxy represents the server.
Internet user → Reverse Proxy → Application servers
Users think they are connecting directly to a website, but the reverse proxy receives the request first and decides which backend server should handle it.
Example:
User → Reverse Proxy → Server A
                     → Server B
                     → Server C
A reverse proxy can:
distribute requests across multiple servers (load balancing),
terminate HTTPS/TLS encryption,
cache static files,
protect backend servers from direct exposure,
block malicious traffic,
route requests to different services.
Think: “Users are coming inward to my application, and the proxy stands in front of it.”

Example with a website
Without a reverse proxy:
Browser → Web Server
With a reverse proxy:
Browser → Reverse Proxy → Web Server
The browser may request:
https://example.com/products
The reverse proxy receives that request first. It may then decide:
/products → Product Service
/login    → Authentication Service
/images   → Cached static-file server
So the proxy becomes a smart “front door” for the system.
In short: a proxy is an intermediary server. A forward proxy works for clients, while a reverse proxy works for servers.
