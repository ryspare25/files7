Forward Proxy vs. Reverse Proxy
1. What is a proxy server, and why is it used?
A proxy server is an intermediary that receives network requests and forwards them to another destination.
Instead of communicating directly:
Client ‚Üí Server
communication goes through an intermediary:
Client ‚Üí Proxy ‚Üí Server
A proxy is used to centralize controls such as:
Security filtering
Authentication
Logging and auditing
Caching
Traffic routing
Hiding IP addresses
Load balancing
TLS/SSL handling
Rate limiting
The important distinction is whose side the proxy represents:
A forward proxy represents the client.
A reverse proxy represents the server.

Implementation & Use Cases
6. When should you use a forward proxy versus a reverse proxy?
Use a forward proxy when the goal is to control, secure, or monitor users going out to the Internet.
Use cases:
Restrict employee Internet access
Block malicious websites
Enforce browsing policies
Log outbound traffic
Hide client IP addresses from external sites
Prevent data exfiltration
Require authentication before Internet access
Control Internet access in schools, offices, or government networks
Use a reverse proxy when the goal is to control, secure, or scale traffic coming into your application.
Use cases:
Put one public endpoint in front of many backend servers
Load balance traffic
Terminate SSL/TLS centrally
Cache static files
Protect backend servers from direct exposure
Add rate limiting
Add a Web Application Firewall
Route requests to microservices
Perform blue-green or canary deployments
A useful interview sentence:
Use a forward proxy to manage outbound traffic from clients, and use a reverse proxy to manage inbound traffic to servers.

9. How does SSL/TLS termination work in a reverse proxy?
Normally, HTTPS encryption is established between a client and a server.
Browser ‚áĄ HTTPS ‚áĄ Application Server
With TLS termination, the reverse proxy handles the HTTPS connection.
Browser ‚áĄ HTTPS ‚áĄ Reverse Proxy ‚áĄ HTTP or HTTPS ‚áĄ Backend Server
Step-by-step flow
The browser connects to https://example.com.
The reverse proxy presents the SSL/TLS certificate.
The browser verifies the certificate.
The browser and reverse proxy establish encrypted communication.
The reverse proxy decrypts the request.
The reverse proxy forwards the request to the backend server.
The backend returns a response.
The reverse proxy encrypts the response before sending it to the browser.
Why terminate TLS at the reverse proxy?
Centralized certificate management
Easier certificate renewal
Reduced encryption workload on backend servers
Simplified application configuration
Easier traffic inspection
Easier routing based on URL, headers, or hostnames
One proxy can serve multiple domains and certificates
Example:
https://example.com/api/users

Reverse Proxy terminates TLS
        ‚Üď
http://internal-api-server:8080/users
Important security consideration
Even if TLS ends at the reverse proxy, traffic between the reverse proxy and backend should often still use encryption, especially when crossing networks, data centers, or cloud environments.
Browser ‚áĄ HTTPS ‚áĄ Reverse Proxy ‚áĄ HTTPS ‚áĄ Backend
This is sometimes called TLS re-encryption or end-to-end encryption.

11. How does a transparent proxy differ from a forward or reverse proxy?
A transparent proxy intercepts traffic without requiring the client to explicitly configure proxy settings.
For example, a user may believe they are connecting directly to a website:
Laptop ‚Üí website.com
But the network silently redirects traffic:
Laptop ‚Üí Transparent Proxy ‚Üí website.com
Key characteristics
Client may not know the proxy exists.
Browser proxy settings may not be required.
The network redirects traffic using routing, firewall, NAT, or policy rules.
Often used in schools, hotels, public Wi-Fi, ISPs, and enterprise networks.

Strong Closing Answer for an Interview
A forward proxy sits in front of clients and controls outbound access to external services. It is commonly used for filtering, privacy, authentication, logging, and data-loss prevention. A reverse proxy sits in front of servers and controls inbound traffic to applications. It is commonly used for load balancing, SSL termination, caching, routing, rate limiting, DDoS protection, and hiding backend infrastructure. The simplest way to remember it is: forward proxy protects clients; reverse proxy protects and scales servers.
